# Contributing

**Evident Technologies LLC**

---

## Overview

This repository is maintained by Evident Technologies LLC. Contributions
are welcome under the terms described below.

## Before You Contribute

1. Read the [GOVERNANCE.md](GOVERNANCE.md) to understand the authority model.
2. Read the [IP_POLICY.md](IP_POLICY.md) to understand intellectual property terms.
3. Check existing issues before opening a new one.

## How to Contribute

### Reporting Issues

- Use GitHub Issues.
- Include: steps to reproduce, expected vs. actual behavior, environment details.
- Do not include credentials, API keys, or personally identifiable information.

### Submitting Changes

1. Fork the repository (external contributors) or create a feature branch.
2. Make focused, atomic commits with clear messages.
3. Ensure all existing tests pass.
4. Submit a pull request against the default branch.

### Pull Request Standards

- One logical change per PR.
- Reference any related issue numbers.
- Include a description of what changed and why.
- Do not modify CI/CD secrets, deployment configs, or governance files
  without prior Manager approval.

## Code Standards

- Follow existing code style and conventions in the repository.
- Do not introduce new dependencies without justification.
- Do not commit secrets, credentials, or environment files.
- Run linters and formatters before submitting.

## Acceptance

All contributions are reviewed by the Manager. Submission of a PR does not
guarantee acceptance. The Manager reserves sole discretion over merges.

By submitting a contribution, you agree to the terms in
[IP_POLICY.md](IP_POLICY.md).

## Prohibited

- Submitting code you do not have the right to license
- Introducing intentional vulnerabilities or backdoors
- Deleting or overwriting governance, security, or license files
- Force-pushing to protected branches

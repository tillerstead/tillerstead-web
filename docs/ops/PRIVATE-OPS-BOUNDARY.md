# Private Ops Boundary

This repository is public and must contain only the public Tillerstead website.

Do not add:

* `admin/`
* `toolkit/`
* `.git-encrypt/`
* private backend code
* subcontractor records
* client/job private data
* secrets
* `.env` files
* generated `_site/`
* `playwright-report/`
* `test-results/`

Future private operational materials belong in a separate private repository, expected name:

`tillerstead/tillerstead-ops`

That repo should be created only after explicit approval.
